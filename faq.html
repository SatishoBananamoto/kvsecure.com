Frequently Asked Questions

Everything you need to know about kv secrets management.

What is kv?

kv is an encrypted secrets manager built for developers who use AI coding tools. It stores your API keys, database URLs, and other sensitive data encrypted with ChaCha20-Poly1305 on your local machine. AI agents can use your secrets through MCP (Model Context Protocol) without ever seeing the actual values.

Is kv free?

Yes. The CLI and MCP server are completely free and open source (MIT license). You get unlimited secrets, unlimited environments, and full encryption at no cost. A paid team plan ($15/team/month) with cloud sync, team management, and CI/CD tokens is coming soon.

What editors does kv work with?

kv works with any editor that supports MCP: Cursor, Claude Code, VS Code, Windsurf, and Cline. Run kv setup cursor (or claude-code / vscode) and it auto-configures the MCP connection.

How does kv keep my secrets safe from AI?

kv uses a 3-tier security profile system:

Safe mode (default): AI can list key names and run commands with secrets injected as environment variables, but never sees the actual values. This is the recommended mode.

Mutate mode (opt-in): AI can also store and delete secrets.

Reveal mode (opt-in): AI can read secret values. Only enable this if you explicitly need it.

The key tool is kv_run — it injects secrets into a subprocess and returns only the exit code. Your AI agent never sees the secret values in chat.

What encryption does kv use?

ChaCha20-Poly1305 AEAD with 256-bit keys — the same cipher used by WireGuard VPN and TLS 1.3. Keys are derived per-environment using BLAKE2b with the environment name as context. The encrypted .enc files are safe to commit to git — only the master key (stored in .secrets/key) must stay private.

Is it safe to commit .enc files to git?

Yes. The .enc files contain only ciphertext — both key names and values are encrypted. Without the master key, they're indistinguishable from random bytes. The master key file (.secrets/key) is auto-added to .gitignore during kv init.

How do I share secrets with my team?

For now, use kv export --key to get a shareable key string (kvkey_...). Send it to your teammate through a secure channel (Signal, 1Password, etc.). They import it with kv import-key kvkey_.... The upcoming team plan will add zero-knowledge cloud sync so sharing is automatic.

Does kv work on Windows?

Yes. kv runs on Windows, macOS, and Linux. It's a pure Python package with one dependency (cryptography) which has prebuilt wheels for all platforms. Install with pip install kv-secrets.

How is kv different from .env files?

.env files store secrets in plaintext — if leaked or committed to git, everything is exposed. kv encrypts everything with military-grade encryption. You can even import existing .env files with kv import .env and they'll be encrypted immediately.

How is kv different from 1Password, Vault, or Doppler?

kv is built specifically for developers using AI coding tools. 1Password is a general password manager ($2.99/mo). HashiCorp Vault and Doppler are enterprise-grade with complex setups. kv is free, zero-config, works offline, and has native MCP integration so your AI editor can use secrets without you pasting them into chat.

Can I use kv without an AI editor?

Absolutely. kv works great as a standalone secrets manager. Use kv set, kv get, kv run from the terminal. The MCP integration is optional — it just makes kv even more powerful when paired with AI tools.

Is kv open source?

The CLI and MCP server are open source under the MIT license. You can inspect every line of code on GitHub. The cloud sync server (for the upcoming team plan) is proprietary.
